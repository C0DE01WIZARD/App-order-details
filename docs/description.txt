Трехуровневая архитектуры, 3 слоя:

    1)Транспортный.
    2)Бизнес.
    3)Базы данных. 


Базовые команды:

1) go run app.go - запуск программы
 

Файлы:
1) launch.json - файл конфигурации
2) README.md - общее описание проекта.

Директории:
1)/cmd - точка входа для нашего приложения
2)/internal - внутренняя логика приложения, бизнес-логика работа с БД
  2.1 /app -точка, где все наши зависимости и логика собираются и запускают приложение. Run-метод, который вызывается из /cmd.
  2.2 /config - Инициализация общих конфигураций приложения, которые мы прописывали в корне проекта.
  2.3 /database (слой базы данных). Файлы содержат методы для взаимодействия с базами данных.
  2.4 /models (слой базы данных). Структуры таблиц баз данных. 
  2.5 /services (бизнес-слой). Бизнес-логика приложения. 
  2.6 /transport (транспортный слой), http-настройки сервера, хендлеры, порты
3) /pkg - библиотеки, используемые в сторонних приложениях.
4) /configs - статические конфигурации нашего приложения, связанные с процессом сборки приложения, yaml-файлы.
5) /api - документация по API. Спецификации OpenAPI или Swagger, файлы JSON Schema, файлы определения протоколов.
6) /build - файлы конфигурации для билда проекта
7) /docs- документирование кода проекта. Документация кода и дизайна (в дополнение к автоматической документации Godoc) храним здесь.
8) /web директория для веб-приложение. Хранятся компоненты для веб-приложений: статические веб-ресурсы, серверные шаблоны и одностраничные приложения.
9) /migrations - миграции, связанные с базами данными
10) 

БД:
1) database/sql - интерфейс для взаимодействия с БД, открыть соединение, выполнить запросы, закрыть соединение
2) go get github.com/lib/pq - установка драйвера 'pq'
3) sudo -i -u postgres && psql - переключение на postgres
